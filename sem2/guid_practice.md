# Гайд по тому, как делать практику
## Начало работы
1. Для того, чтоб успешно выполнить задания по практике нужно для начала взять шаблон, в котором пишется отчет. Шаблон [тут](https://www.overleaf.com/read/cjtjbmnbgqrh#6a9cb3)
2. Нужно скопировать себе этот шаблон (для дальнейшего редактирования)
3. Выбрать предметную область, которую вы будите дополнять. Суть практики - дополнить стандарт новыми знаниями, понятиями, новой информацией.
4. После того, как вы выбрали свою область, нужно искать статьи, книжки, различные публикации по вашей теме. И все, что вы нашли - формализовывать в отчете.
## Описание шаблона
В самом шаблоне представлены разлиные категории 

<img width="288" alt="Снимок экрана 2024-05-26 в 22 38 24" src="https://github.com/iis-32170x/RPIIS/assets/145375460/ed3fe19e-93f3-4079-9d39-88ff22a27ffb">

* images - папка, в которой будут хранится все картинки, которые нужны вам для отчета
* conclusion.tex - файл, в котором вы напишите заключение свой работы (краткий вывод)
* content.tex - так скажем самый важный файл, который соединяет все остальные файлы в себе
* section 1/2/3 - файлы, в которых вы будите писать все, они разбиты на секции (0,1,2,3....).
* title - титульник, который нужно отредактировать под себя ( ФИО, номер группы, ФИО преподавателя, год)
* bibliography - ваш список литературы.
* section 0 - постановка задачи, ее надо будет отредактивоть под свой файл (лучше сделать в конце).

Остальные файлы не нужно редактировать, это библиоетки, чтоб ваш шаблон работал:)

Всю свою работу по формализации нужно разбить на логические части, чтоб было удобнее пользоваться и ориентироваться. Каждая часть будет подключатся в этом файле. Например:

* /newpage - новая старница 
* /section{} - в скобках будет писаться название ващей главы
* /input{} - подключение конкретного .tex файла, в котором вы описываете данную главу

Благодаря командам **\scnheader {}** и **\scnidtf{}** мы смогли ввести наше понятие. Для каждого понятия будет такой вид, для каждой области, которую мы описываем будет использоваться **\scnheader {}**. Если у понятия есть несколько вариаций, их просто вот так "друг за другом" и пишем.

Все понятия, НОВЫЕ понятия, которые мы вводим, описываются так. Поэтому все, что у меня в статье упоминается из понятий будет иметь вид:

С помощью этих команад 
```
\begin{scnrelfromset}{}
\scnitem{}
\end{scnrelfromset}
```

в латехе мы можем дополнять наше определение, чем захотим. Например писать *недостатки этой системы* и перичеслять, *текущее состояниие* и перечислять, *рассматриваемые вопросы* и перичислять. Вообщем очень много "дополнений".

Дальше я хочу написать класиффикацию экспертных систем, там они и перичисляются, и даются определения. Вот как это сделать:

<img width="1433" alt="Снимок экрана 2024-05-26 в 23 55 18" src="https://github.com/graffegiraffe/kater/assets/145375460/9df3d8ce-f4aa-403e-9cc5-52a86cfa38da">

Используя этот блок команд 
```
\scnheader{Классификация экспертных систем}
\begin{scnsubdividing}
   \scnitem{Интерпретация данных}
        \begin{scnindent}
            \scnidtf{Процесс определения смысла данных, результаты которых должны быть корректными и согласованными}
        \end{scnindent}
    \scnitem{Диагностика}
      \begin{scnindent}
            \scnidtf{Процесс обнаружения неисправностей в некоторой системе. Под «неисправностью» понимается отклонение от нормы.}
        \end{scnindent}
\end{scnsubdividing}
```

Далее я хочу расписать покаления ЭС, это можно сделать с помощью команд:
```
\scnheader{Поколения экспертных систем}  
\begin{scnrelfromlist}{Различают следующие поколения ЭС}
\scnitem{Первое поколение ЭС}
 \scntext{пояснение}{Характеризует повторяющийся логический вывод эксперта. Этому поколению свойственен ряд ограничений: узкая область экспертизы, в основном используется вид личностных знаний эксперта, существует проблема модификации ЭС в процессе ее эксплуатации.}
\scnitem{Второе поколение ЭС}
 \scntext{пояснение}{Характеризует появление так называемых «оболочек» ЭС, которые предназначены для автоматического получения знаний из внешней среды.}
\scnitem{Третье поколение ЭС}
 \scntext{пояснение}{Характеризует появление гибридных или интегрированных ЭС, которые способны самообучаться и адаптироваться в процессе функционирования непосредственно в самой системе.}
\scnitem{Четвертое поколение ЭС}
 \scntext{пояснение}{Характеризует появление динамических систем, которые имеют ярко выраженные адаптивные свойства, динамизм БЗ и подсистему логического вывода.}
\scnitem{Пятое поколение ЭС}
 \scntext{пояснение}{Характеризует появление распределенных сетей из совместно функционирующих систем. Особенностью этого поколения является синтез нескольких ЭС, когда решение одной ЭС выступает в качестве исходных данных для другой.}
\end{scnrelfromlist}
```

Но смотря на последний скриншот, мы видим что там появилось *пояснение*. Его оформить можно используя команду **\scntext{пояснение}{ваш текст}** ( в первой скобке можно писать примечание, пояснение и тд. Когда нужно пояснить что-то, опрделение, список и тд., используйте эту команду.
